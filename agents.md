# Agent Instructions

Each branch that includes code changes must include one appropriate app version bump. Do not bump the version again for every iteration, follow-up fix, or review adjustment on the same branch.

- For fixes, maintenance work, and other non-feature changes, bump the patch version.
- For new user-facing features or meaningful product capability changes, bump the minor version.
- If the branch has already been bumped, keep that single bump unless the final branch scope changes enough to require a different bump level.
